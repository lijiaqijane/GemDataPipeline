# FireCrawl DeepSeek 适配修改总结

### 1. `lib/generic-ai.ts`
源代码已包含 `OPENAI_MODEL_MODE` 判断逻辑（`.chat()` 优先），无需修改。
*运行时发现预构建 Docker 镜像的编译版本缺失此逻辑，通过启动补丁修复。*

### 2. `lib/extract/fire-0/llmExtract-f0.ts`

- 添加 `NoObjectGeneratedError` 导入
- 添加 `isDeepSeekModel()` 函数
- 添加 `safeParseJSON()` 函数
- DeepSeek 主动检测 → `generateText` + JSON 提示词
- 增强 `experimental_repairText`：优先 `safeParseJSON`
- 增强 `NoObjectGeneratedError` 处理

### 3. `scraper/scrapeURL/transformers/llmExtract.ts`
同上，针对通用 extract 路径。

## 持久化启动补丁（使修改在容器中生效）

Docker 使用预构建镜像 `ghcr.io/firecrawl/firecrawl:latest`，TypeScript 源码修改无法直接生效。通过以下文件在启动时自动修补：

### 补丁文件

| 文件 | 作用 |
|------|------|
| `docker/helpers.js` | `isDeepSeekModel()` + `safeParseJSON()`，被复制到 `dist/lib/deepseek-helpers.js` |
| `docker/patch.sh` | 启动脚本：拷贝 helpers + `sed` 修补 `generic-ai.js` |

### 修补内容

| 容器内文件 | 修改 |
|-----------|------|
| `/app/dist/src/lib/generic-ai.js` | 将 `o3-mini` 专属的 `.chat()` 条件去除，**对所有 OpenAI 兼容模型使用 Chat Completions API**。这是最关键的修复 — DeepSeek/Volcengine 仅支持 Chat Completions，不支持 Responses API |

### 持久化机制

`docker-compose.yaml`：
```yaml
volumes:
  - ./docker:/docker
command: sh -c "sh /docker/patch.sh && node dist/src/harness.js --start-docker"
```

## 关键发现

**Responses API 不兼容是核心问题**：`@ai-sdk/openai` v2.x 默认使用 Responses API（`POST /responses`），DeepSeek/Volcengine 只支持 Chat Completions API（`POST /chat/completions`）。修复前：AI SDK 收到 DeepSeek 响应后因缺少 `annotations` 字段抛出 `AI_TypeValidationError`。
